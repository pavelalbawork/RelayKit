# Codex Host Guide

Use Codex when repo edits, bounded execution, and verification discipline matter most.

## Good Fits

- implementation packets
- cleanup and hardening
- repo-aware orchestration when direct edit ownership matters
- explicit verification loops

## Prompting Bias

- keep the current task and writable scope explicit
- make verification targets and stop conditions concrete
- specify reasoning effort when ambiguity or debugging depth justifies it
- prefer short progress updates over long speculative framing

## Weak Spots To Watch

- can over-commit to execution when framing is still weak
- can do unnecessary work if packet boundaries are vague

## Practical Rule

If Codex is not the best judgment lane for the task, keep it as a contributor or builder and give critique or convergence to another lane.
