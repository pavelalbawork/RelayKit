# Claude Code Host Guide

Use Claude Code when framing, orchestration quality, critique, or convergence is the bottleneck.

## Good Fits

- routing and orchestration
- packet design and decomposition
- second-opinion critique
- synthesis across several partial outputs

## Prompting Bias

- use clean sections for context, constraints, deliverables, and stop rules
- separate planning from execution intent
- tell Claude when to stay advisory versus when to own a gate
- keep lane relationships explicit when more than one tool is active

## Weak Spots To Watch

- can spend too many tokens on framing when execution is already clear
- should not be the default mechanical execution lane without a reason

## Practical Rule

Use Claude for leverage, not ceremony. If the work is already execution-ready, hand the packet to a contributor lane quickly.
