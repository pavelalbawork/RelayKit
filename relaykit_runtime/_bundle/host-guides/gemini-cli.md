# Gemini CLI Host Guide

Treat Gemini CLI as a real host, not just a dead-end backup.

## Good Fits

- overflow execution
- lower-cost parallel lanes
- lightweight research, drafting, or verification packets
- alternative model perspective when another host is saturated

## Prompting Bias

- keep prompts compact and labeled
- define output format early
- use bounded packets with explicit next-step expectations
- choose Pro versus Flash intentionally instead of by habit

## Weak Spots To Watch

- broad or under-specified packets can waste turns quickly
- should not inherit orchestration responsibility by accident

## Practical Rule

Use Gemini CLI to widen parallel capacity or reduce spend, but still give it the same packet discipline as premium lanes.
