# Positioning

## Category

`Harness Augmentation`

## Core Narrative

RelayKit augments existing harnesses like Codex, Claude Code, and Antigravity so one operator can run multi-tool work in parallel without giving up control.

The main story is:

- multi-tool execution
- human-in-the-loop control
- deliberate parallelization
- harness-specific role assignment
- augmentation instead of replacement

## Product Role

`RelayKit` is a harness-augmentation product for operators who want a focused, productized surface for multi-tool parallel execution under human control.

## Supporting Mechanisms

These are mechanisms, not the top-level identity:

- task intake
- setup recommendation
- checkpoints
- handoffs
- host onboarding
- personas

## Messaging Rule

Lead with:

`Harness augmentation for multi-tool, human-in-the-loop parallel execution.`

Then explain:

`RelayKit makes existing harnesses work as coordinated lanes under human control instead of treating every tool as a generic assistant.`
